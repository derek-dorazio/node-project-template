# Requirements

This directory contains product requirements that drive the spec-driven development lifecycle.

## Expected Contents

### Product Requirements
- **product-requirements.md** — What the product does, who it serves, what problems it solves.

### Domain Model
- **domain-model.md** — Domain objects, relationships, lifecycle states, ownership boundaries.

### Module Map
- **module-map.md** — How domain objects are grouped into service modules with clear boundaries.

### User Roles and Permissions
- **roles-and-permissions.md** — User identity model, roles, permissions, authorization boundaries.

## Workflow

These documents are created during **Phase 1** of the spec-driven lifecycle (see `rules/workflow-rules.md`). They inform the use-case companions in `plans/` and the design plans that follow.

Requirements should be treated as living documents — update them when product direction changes, but do not let them drift silently from the implemented reality.
